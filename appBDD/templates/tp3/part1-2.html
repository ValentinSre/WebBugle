Composition d'une page

Chaque élément du Web est défini par ce que l'on appelle une URI.



URI

L'URI (Uniform Resource Identifier) permet d'identifier une entité utilisée sur le web. Les URI peuvent être reliées entre elles.

Il s'agit d'une extension de l'URL (qui permet d'identifier les pages du web uniquement). Ainsi, tout ce qui est identifié par une URI est une ressource du web accessible sur le web.
Cela rassemble donc les documents du web (identifiés par leur URL) mais aussi toutes les choses dont on parle sur le web, par exemple une table, une voiture, un humain ou encore une idée, un concept abstrait... Si l'on schématise, l'URI correspond à la concaténation entre l'URL pour localiser une ressource et de l'URN (Uniform Resource Name) qui est un identifiant unique pour une ressource.


Exemple d'URI

Plaçons nous dans le contexte d'une bibliothèque, à la recherche d'un livre particulier. La bibliothèque représente le Web, le livre représente une donnée recherchée.

Si l'on demande Amazing Spider-Man N°1 au bibliothécaire, il ne saura pas spécifiquement quel livre nous souhaitons (plusieurs livres portent ce titre), il ne saura donc pas où chercher. Nous ne pourrions pas avoir le livre (simplement).

Identifier le livre

Pour identifier un livre, il existe un identifiant unique, appelé ISBN. Ce code désigne un livre particulier (au sens large) et permet donc d'identifier quel livre Amazing Spider-Man N°1 nous souhaitons.

Par analogie, il s'agit de l'URN du livre : nous pouvons désormais identifier le livre avec précision.

Localiser le livre

Pour localiser le livre dans la bibliothèque, il faut savoir dans quel rayon le chercher et sur quelle étagère. Si l'on connaît ces paramètres, on peut savoir où se trouve le livre dans la bibliothèque.

Par analogie, il s'agit de l'URL du livre : nous pouvons le localiser avec précision.

Le rayon pourrait être comparé au site Web, l'étagère pourrait être comparée à la page du site.


Indépendamment, l'ISBN ne permet pas de trouver le livre et l'emplacement du livre ne permet pas de savoir quel livre nous cherchons. Combinés, ils permettent de tout savoir du livre : ils forment l'URI du livre.



Comprendre l'URI

Vérifie ta bonne compréhension de ces notions.


Résumé

L'URL permet de localiser une ressource dans le web.
L'URN permet de référencer une ressource sur le web sans la localiser.
L'URI est l'ensemble de ces deux paramètres.
